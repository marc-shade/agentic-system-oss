# Save information to knowledge graph

Use enhanced-memory MCP to store important information for future sessions.

## What gets saved
- Technical learnings and patterns discovered
- Project-specific context and decisions
- Problem-solving approaches that worked
- Important code patterns and architectures

## How to use
After completing a significant task or learning something valuable, this command stores it in persistent memory.

The information will be:
1. Compressed for efficient storage
2. Indexed for semantic search
3. Versioned for history tracking
4. Available across sessions

## Memory Types
- **working**: Short-term, high-priority items
- **episodic**: Time-bound experiences and events
- **semantic**: Timeless concepts and knowledge
- **procedural**: Skills and procedures

---

Save the following to memory with appropriate categorization:

$ARGUMENTS

Use mcp__enhanced-memory__create_entities to store this information with:
- Descriptive entity name
- Appropriate entity type (learning, pattern, decision, etc.)
- Detailed observations capturing the key information
